## O que foi implementado

### 1. ApkgBuilderService
- `build_apkg_from_session()`: SessionRecord → .apkg file (genanki)
- `_build_deck()`: Cria Anki deck com modelo customizado
- `_hash_to_deck_id()`: Deterministic deck ID (idempotency)
- `_extract_subject()/_extract_topic()`: Parse review_card metadata
- Suporta genanki 0.13.1 (write_to_file API)

### 2. WeeklyReportJobService
- `generate_weekly_report()`: Aggregated stats para período
- `aggregate_session_stats()`: List[SessionRecord] → metrics
- **Privacy-first**: Nenhum telegram_id em outputs
- WeeklyStats dataclass com to_dict()
- ReportExporter para JSON + text formats

### 3. APKG Build Result
- ApkgBuildResult dataclass para result tracking
- Success/failure states com error messages
- Serializable para logging/persistence

## Arquitetura

```
Study Session (queued_local_build)
  ├─ ApkgBuilderService
  │   ├─ extract subject/topic
  │   ├─ create genanki Deck
  │   ├─ build note from review_card
  │   └─ write .apkg file (deterministic)
  └─ WeeklyReportJobService
      ├─ fetch sessions
      ├─ aggregate stats
      ├─ **filter telegram_id** (privacy)
      └─ export (JSON/text)
```

## Privacy Guarantees
- ✅ WeeklyStats.to_dict() não contém telegram_id
- ✅ ReportExporter nunca expõe telegram_id
- ✅ Aggregation por mood/subject/topic (não por usuário)
- ✅ All session IDs em reports são anônimos
