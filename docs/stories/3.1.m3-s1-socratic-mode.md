## Summary
Implementar modo socrático (Socratic Questioning) — quando aluno erra, faz até 2 perguntas guiadas antes de revelar a resposta correta e explicação.

## Acceptance Criteria
- [x] Mood check-in: se "cansada" → explicação direta; senão → socrático
- [x] Primeira pergunta socrática (Q1) gerada com templates
- [x] Resposta capturada e transitada para Q2
- [x] Segunda pergunta socrática (Q2) gerada com templates
- [x] Após Q2 → explicação final + DONE
- [x] Integration com review_card (M2-S3) preservada
- [x] Tests cobrem ambos os caminhos (**6 testes, 60 total**)
- [x] States corretos: DONE (tired) ou WAITING_SOCRATIC_Q1 → Q2 → DONE

## Implementation Files
- [x] `app/services/socratico_service.py` - lógica de perguntas socrática (110 linhas)
- [x] `app/services/me_testa_answer_service.py` - roteamento para SocraticoService
- [x] `app/services/me_testa_service.py` - handlers para Q1 e Q2
- [x] `app/api/runtime.py` - injeção de SocraticoService
- [x] `tests/test_socratic_mode.py` - 6 testes end-to-end

## O que foi implementado

### 1. SocraticoService (app/services/socratico_service.py)
- `route_incorrect_answer()`: Avalia mood e roteia
  - Se "cansada" → `skip_to_direct_explanation()` → DONE
  - Senão → `generate_q1()` → WAITING_SOCRATIC_Q1
- `process_q1_response()`: Captura resposta Q1, gera Q2
- `process_q2_response()`: Captura resposta Q2, entrega explicação
- Template-based Q1 e Q2 (ready para upgrade para LLM)

### 2. Integração com Me-Testa
- `MeTestaAnswerService._handle_incorrect_answer()` agora chama `SocraticoService`
- `MeTestaService.handle_event()` adiciona handlers para WAITING_SOCRATIC_Q1/Q2
- Error classification (M2-S3) preservado em `metadata.review_card`
- Review card gerado antes da rota socrática

### 3. Testes (6 novos, 60 total)
- **SocraticModeNotTiredTest**: Full flow Q1 → Q2 → DONE
- **SocraticModeTiredTest**: mood="cansada" → direct → DONE
- **SocraticIntegrationTest**: Error classification + follow-ups
- **SocraticServiceDirectTest**: Unit tests de Q1/Q2/direct

### 4. Flow Diagram
```
Resposta Incorreta
  ├─ Classificar erro
  ├─ Preparar review_card
  ├─ [M3-S1] route_incorrect_answer()
  │   ├─ mood="cansada" → skip_to_direct_explanation() → DONE
  │   └─ mood≠"cansada" → generate_q1() → WAITING_SOCRATIC_Q1
  │       └─ Q1 resposta → process_q1_response() → WAITING_SOCRATIC_Q2
  │           └─ Q2 resposta → process_q2_response() → DONE
```
